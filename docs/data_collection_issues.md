# 数据采集问题总结

## 问题1: P0标的池缺失1个 ✅ 已定位

**现象**: 配置47个标的，实际入库46个

**原因**: 恒生指数 `HSI.HI` 未采集成功

**可能原因**:
1. Tushare对港股指数的支持有限
2. 需要特殊权限或不同的接口
3. 代码格式不正确（应该用 `HSI` 而不是 `HSI.HI`）

**解决方案**: 
- 测试正确的恒生指数代码
- 或使用港股ETF作为替代（如159920.SZ恒生ETF）

## 问题2: 4个中证指数历史数据不足 ✅ 已确认

**现象**: 
- 000932.CSI (中证消费) - 仅23条，从2026-04-24开始
- 000933.CSI (中证医药) - 仅23条，从2026-04-24开始  
- 000934.CSI (中证金融) - 仅23条，从2026-04-24开始
- 000935.CSI (中证800有色) - 仅23条，从2026-04-24开始

**原因**: Tushare数据源限制，这些指数在2026年只有4月24日之后的数据

**验证**: 直接调用Tushare API确认返回相同结果

**影响**: 不影响系统功能，这是数据源本身的限制

**解决方案**: 
- 接受现状（数据源限制）
- 或使用对应的ETF数据作为替代

## 问题3: 测试脏数据 ✅ 已清理

**现象**: 510300.SH有2024年的测试数据

**原因**: 之前测试时插入的数据未清理

**解决**: 已执行 `DELETE FROM daily_bars WHERE symbol='510300.SH' AND trade_date < '2026-01-01'`

## 问题4: 限速统计Bug ✅ 已修复

**现象**: 总等待时间始终为0

**原因**: `wait_time` 变量在批次冷却时被覆盖，未累加

**修复**: 改用 `total_wait` 变量累加所有等待时间

## 问题5: 重复写入Bug ✅ 已修复

**现象**: 每次运行都追加数据，不检查已有数据

**原因**: 未在写入前检查数据库最新日期

**修复**: 
- 增量模式下查询 `max(trade_date)`
- 如果已是最新则跳过，打印 `SKIPPED`
- 日志清晰显示：数据库最新日期、目标结束日期、是否需要更新
